================================================================================
ARCHITECTURE HIGHLIGHTS
================================================================================

Clean modular design:
  - Core: State management, action registry, logging
  - Actions: Transformations and models (easily extensible)
  - UI: Dash interface with callbacks
  - Utils: Column analysis and helpers

Key design patterns:
  - BaseAction abstract class (all actions follow same pattern)
  - ActionRegistry (central registration)
  - StateManager (single source of truth)
  - Smart suggestions (context-aware UX)

Benefits:
  - Easy to add new actions (15-30 min each)
  - Easy to test (actions are independent)
  - Easy to understand (clear separation of concerns)
  - Easy to extend (plugin architecture)
